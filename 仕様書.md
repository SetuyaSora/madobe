# 仕様書：Chrome Wallpaper

## 1. システム構成・拡張機能構造
本機能は、Chromeの新規タブ（New Tab）ページをカスタムHTMLにオーバーライドするChrome拡張機能（Manifest V3）として構築します。

### ディレクトリ構成
```text
chrome-wallpaper/
├── manifest.json         # 拡張機能のマニフェストファイル
├── newtab.html           # 新規タブのメインHTML
├── newtab.js             # メインロジック (ES Modules エントリーポイント)
├── js/                   # JavaScript 機能モジュール群 (ES Modules)
│   ├── state.js          # アプリケーション共通状態・定数・要素キャッシュ

│   ├── physics.js        # グリッド衝突判定＆多方向連鎖退避アルゴリズム
│   ├── widgets.js        # 各種ガジェット (時計・カレンダー・付箋Todo) の描画・操作
│   ├── contextmenu.js    # 右クリック不透明度調整メニュー制御
│   ├── suggest.js        # Google suggest ＆履歴の統合検索制御
```

## 2. 機能要件

### 2.1. 背景動画再生機能
- **ビデオ要素 (`<video>`)**:
  - 画面全体にフィット（`object-fit: cover`）させ、背面（`z-index: -2`）に配置する。
  - ループ再生 (`loop`)、自動再生 (`autoplay`)、初期ミュート (`muted`) を有効にする。
- **壁紙ソース切り替え**:
  - **プリセット（デフォルト）**: 拡張機能内に同梱された `assets/default_bg.mp4` を再生。
  - **外部URL**: ユーザーが指定した動画URL（例: HTTPS経由のMP4/WebM）を再生。URLは `chrome.storage.local` に保存し、次回起動時も自動ロードする。
  - **ローカルファイル**: 
    - ユーザーのPCからファイルを選択して再生。
    - **永続化 (IndexedDB)**: 選択された動画データ（Blobオブジェクト）は、ブラウザ内部のデータベースである `IndexedDB` に一意のキーで永続保存される。これにより、新規タブの再読み込みやブラウザ再起動後もリセットされず、自動的に再生が再開される。
    - **容量とパフォーマンス**: 巨大な動画ファイルは読み込み遅延やブラウザの容量上限に達する恐れがあるため、「容量目安 50MB 以下の軽量動画を推奨」する旨をUIに明記する。
    - **ディスク容量の最適化**: ユーザーが壁紙ソースを「デフォルト」または「外部URL」に切り替えた場合、不要なディスクスペース消費を防ぐため、IndexedDB から保存済みの動画データを自動的に削除（クリア）する。

### 2.2. パフォーマンス・軽量化要件 (「軽いこと」の実現)
- **タブ非アクティブ時の再生停止**:
  - `document.visibilityState === 'hidden'`（別タブの閲覧中、またはウィンドウ最小化時）を検知し、動画の再生を一時停止（`video.pause()`）する。
  - タブが再度アクティブになったら再生を再開（`video.play()`）する。
- **CSS最適化**:
  - ビデオ要素に対して `will-change: transform` または `transform: translateZ(0)` を適用し、GPUアクセラレーションを有効化する。
- **軽量アセット**:
  - 内蔵するデフォルト動画（`default_bg.mp4`）は、高圧縮でデコード負荷の低いMP4/WebMフォーマットを使用する。

### 2.3. 新規タブ画面 UI・コンテンツ
- **検索バー (ウィジェット化仕様)**:
  - **ウィジェットとしての実装**:
    - 画面中央への固定配置を廃止し、**独立したウィジェット (`type: 'search-bar'`)** として動作させる。
    - 初期状態（デフォルト設定）では画面中央付近（幅10マス、高さ1マス）に最初から1つ自動配置される。
    - ユーザーは検索バーのヘッダーをドラッグして画面の好きな場所にスナップ配置し、右下ハンドルで横幅を伸縮させることができる。
    - 検索バーを削除（×）することも可能。削除した場合は、左端ドロワーのウィジェットタブから「Google検索」をドラッグ＆ドロップすることで画面に再配置できる。
    - 検索バーへの入力時に出現するサジェストリスト（予測・履歴候補）は、検索バーの移動先に追従して常に検索バーの直下に絶対配置で展開される。
- **クイックリンク (ショートカット) - ホバードロワー仕様**:
  - **非表示ドロワーの導入**:
    - 画面中央の表示を廃止し、動画の視覚的魅力を最大化するために**画面左端のホバードロワー**内に格納する。
    - 通常は画面外（`transform: translateX(-100%)`）に隠れており、マウスカーソルを画面の左端に近づけることで、左から滑らかにスライドインして出現する。
    - すりガラス調（`backdrop-filter: blur(24px)`）の美しい縦型ドロワーとしてデザインする。
    - ドロワー内には、縦スクロール可能なショートカット一覧および「新規追加（＋）」ボタンを配置。
    - ユーザーが新しいリンクを追加、既存のリンクを編集・削除できる。
    - **フォーカス維持設計**: ショートカットの追加/編集ダイアログが開いている間は、マウスがドロワー領域外へ出ても、入力の妨げにならないようにドロワーは開いた状態を維持する。
    - リンクデータは `chrome.storage.local` に保存し永続化。
- **ウィジェット管理システム (デスクトップガジェット)**:
  - **ドロワーのタブ化**:
    - Left端ドロワーの上部にタブ切り替えボタンを設置し、「ショートカット」と「ウィジェット」を切り替え可能にする。
    - 「ウィジェット」タブ内には、ドラッグ可能な6種のウィジェット見本カード（デジタル時計、アナログ時計、カレンダー、メモ帳、Google検索、RSSフィード）を表示する。
  - **表示モード（通常モードと編集モード）**:
    - **通常モード（通常表示）**: ウィジェットの外枠、ヘッダー、削除ボタン、リサイズハンドルを非表示にし、中身のコンテンツのみが背景動画の上に直接浮かぶ。通常表示時の文字には視認性を保つため、うっすらテキストシャドウを付与する。通常時はウィジェットの移動やリサイズは行えない。
    - **長押し（ロングプレス）起動**: ウィジェットを約0.7秒間（700ms）マウスで押し続けることで「編集モード」が起動する。
    - **編集モード（Wiggleエフェクト）**: 編集モード中はすべてのウィジェットがすりガラス枠をまとい、ヘッダーとリサイズハンドルが出現する。また、iPhoneのようにプルプルと左右に小さく震えるアニメーション（Wiggleエフェクト）が適用される（操作中のウィジェットは操作性確保のため一時的に震えが停止する）。
    - **編集モードの終了**: 画面の何もない場所（背景）をクリックすると、編集モードが終了し通常モードへ戻って固定される。
  - **重なり防止とiPhone風位置入れ替え（ポジション・スワップ）**:
    - **ドラッグ中のリアルタイムスワップ**: 
      - ドロップする前（ドラッグ操作の途中）であっても、ウィジェットを重ねた瞬間に、重なられた既存ウィジェットがドラッグ元の空いた位置へ「スッと滑らかにアニメーションスライド」して入れ替わる。
      - スワップが発生するたびに、空いたドラッグ元座標がローリング追従され、連続した入れ替えが可能。
      - ドラッグ中の本人以外のウィジェットは CSS Transition により滑らかにスライド移動するが、ドラッグ中の本人には `transition: none` を適用してマウス追従の遅延を完全にゼロ（高レスポンス）にする。
    - サイズ違い等の影響で、入れ替わったウィジェットが第三のウィジェットと衝突した場合は、玉突きのように全方向（上下左右）の最も移動距離が少なくて済む空き領域へ自動スライド退避する。
  - **初期ウィジェットの種類**:
    - **Google検索**: 予測サジェストと閲覧履歴候補を動的表示する検索バー。
    - **デジタル時計**: 美しい大きなフォント of デジタル時刻・日付表示。縦1マスのスリム表示に対応（最小 3x1 マス）。
    - **アナログ時計**: CSSアニメーションで回転する時針・分針・秒針を持つ円形時計。コンパクト表示に対応（最小 2x2 マス）。
    - **カレンダー**: 今月のカレンダー（日曜日〜土曜日のグリッド、今日の日付をハイライト）。
    - **メモ帳 (Todo)**: テキストを自由に書き込める付箋紙風ボード。入力内容は自動で永続保存される。超小型リサイズに対応（最小 4x4 マス）。入力されたテキスト of 改行数（高さ）に応じてウィジェット自身の縦幅（`gridH`）が自動的に伸び縮みするオートフィット機能、および他ウィジェットとのリアルタイム衝突回避スライド退避が連動する。
    - **ToDoリスト (タスク管理)**:
      - チェックボックス付きリスト形式のタスク管理ウィジェット（最小サイズ 6x4 マス）。
      - **完了切り替え**: 各タスクの左端にある丸型チェックボタンをクリックすることで「未完了/完了（打消し線・半透明化）」を切り替え。
      - **グラデーション進捗バー**: ウィジェット上部に、全タスクに対する完了タスクの割合（%）を示す美しいグラデーション進捗バーをリアルタイム描画。
      - **ステータスフィルター**: 「すべて」「未完了」「完了済み」の切り替えタブがあり、クリックで表示されるタスクを動的に絞り込み。
      - **タスクの追加・削除**: 下部の入力ボックスからタスク名を入力し、Enterキーまたは「＋」ボタンでリストに追加。各タスクの右端の×ボタンで個別削除。
      - データは各ウィジェット設定内に `todos` 配列（`{ id, text, completed }`）として保存され、自動で永続化される。
    - **RSSフィード**: 
      - **時系列マージソート**: 1つのウィジェットに改行区切りで複数のRSS/AtomフィードURLを登録可能。各フィードから非同期・並列でデータを取得（`Promise.allSettled` によるフォールバック）し、すべての日付（公開日時）を基準に新しい順（降順）でマージソートして表示（最大30件）。
      - **配信元の明記**: ティッカーおよびリストにて、どのフィードから取得されたニュースかが判別できるよう、記事タイトルの先頭に `[Yahoo]` や `[ITmedia]` のように配信元チャンネル名（ソース名）タグを表示する。
      - **動的通信許可（CORSバイパス）**: 右クリックメニュー内のURLリストに入力された複数のドメインに対し、ブラウザへの通信許可ダイアログ（`chrome.permissions.request`）を1回でまとめて一括申請可能。
      - **等速スクロールと速度調整**: ニュースの総文字数（長さ）に関わらず、流れる物理速度（秒速ピクセル数）を常に一定に保つ等速計算をJavaScriptで動的に実行。速度調整は `速度レベル 1 (遅い)` 〜 `10 (速い)` （デフォルト 5）の直感的なスライダーで設定可能。
  - **ドラッグ＆ドロップによる追加**:
    - ドロワー内の見本カードを画面中央にドラッグ＆ドロップすることで、新規ウィジェットを背景に追加できる。
  - **48x24グリッドスナップ**:
    - 画面全体を「横48マス×縦24マス」の仮想グリッドに分割（細分化による緻密なレイアウト調整に対応）。
    - 移行互換性として、過去の 24x12 基準の保存データを初回起動時に自動で「位置・サイズすべて2倍」にスケーリング変換（マイグレーション）する。
    - ウィジェットの追加、マウスでの位置移動、右下のつまみを掴んだリサイズ（拡大縮小）操作は、すべて最寄りのグリッドセルに「ピタッ」と吸い付くように自動整列する。
    - 位置・サイズデータ（`gridX`, `gridY`, `gridW`, `gridH`）はパーセンテージ換算されるため、ウィンドウの拡大縮小時にもレイアウトが壊れないレスポンシブ動作を保証する。
  - **削除機能**: 各ウィジェットのヘッダーにある削除（×）ボタンでいつでも撤去可能。
  - **データの永続化**: 配置状態は `chrome.storage.local` に配列として保存され、タブを開くたびに完全復元される。
- **設定パネル (オーバーレイ)**:
  - 画面の右下に配置されたギアアイコン（SVG）をクリックすることで開閉。
  - グラスモルフィズムデザインを採用した、モダンで半透明なパネル。
  - **設定項目**:
    - 壁紙ソースの選択（デフォルト / 外部URL / ローカルファイル永続再生）
    - 外部URL入力欄（保存ボタン付き）
    - ローカルファイル選択ボタン
    - 音量コントロール（0%〜100%、スライダー）
    - 再生速度（0.5x, 1.0x, 1.5x, 2.0x、ラジオボタンまたはセレクトボックス）
    - 壁紙の不透明度（0%〜100%のスライダー、動画の上に重ねる暗色オーバーレイの透明度を調整し、検索バーなどの文字の視認性を確保する）
    - （表示項目のオンオフトグルはUIクリーンアップのため廃止）
    - **フォントファミリーの変更 (ハイブリッド方式)**: 
      - 設定パネル内にフォント選択用のドロップダウンリストを設置。Inter(デフォルト), Noto Sans JP, Noto Serif JP, M PLUS Rounded 1c, Shippori Mincho, JetBrains Mono のプリセットから簡単に切り替え可能。
      - リストの最下部に「カスタム (Google Fonts)...」を設け、これを選択した時のみ、任意の Google Fonts 名を直接入力できるテキストボックスが出現する。
      - 指定されたフォント名は拡張機能の `storage` に保存され、起動時および変更時に Google Fonts API（`https://fonts.googleapis.com/css2...`）から動的にインポートされた上で、画面全体のベースフォント（CSS変数 `--font-main`）として適用される。

## 3. 非機能要件・デザインシステム
- **デザイン・美観**:
  - システム絵文字は使用せず、Lucide Icons（SVGパス）またはCSSによるモダンなアイコンを採用。
  - フロントフォントは Google Fonts（Inter等）をインポートしてモダンな印象にする。
  - ダークモードを意識したスタイリッシュな半透明（グラスモルフィズム）デザイン。
- **レスポンシブ**:
  - さまざまな画面解像度（ノートPC、デスクトップ、4Kモニターなど）に対応。

## 4. 開発環境・API仕様
- **Chrome Extension Manifest**: `manifest.json` (Version 3)
- **パーミッション (`permissions`)**:
  - `storage` (設定情報およびショートカットリンクの保存用)
  - `history` (閲覧履歴 of 取得用)
- **オーバーライド設定**:
  - `chrome_url_overrides` を使用し、`newtab` ページを `newtab.html` にマッピング。
